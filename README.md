# Replication Package

**Adopting the Language of the Ethnic Majority: Evaluation of the Philippines' Bilingual Education Policy**

Takiko Igarashi (University of Tokyo) and Kazushi Takahashi (GRIPS)

## Overview

This repository contains the Stata do-files and R scripts used to construct
the analysis dataset and produce the tables and figures in the paper. It does
**not** include the underlying microdata, which must be obtained separately
from the Philippine Statistics Authority (PSA).

## Data Access

This study uses the 2020 Philippine Census of Population and Housing (CPH),
collected and maintained by the Philippine Statistics Authority (PSA). The
data are publicly disclosed by PSA and available upon request through its
data repository/archives: https://psada.psa.gov.ph/home

Elevation control data are derived from SRTM (30 arc-second / 90m)
elevation data; see `r_scripts/` for extraction code.

## Requirements

- Stata 17 or later
- Packages: `reghdfe`, `coefplot`, `estout` (for `esttab`)
- Additional packages (`did_multiplegt`, `spmap`) may be required by the
  data construction do-files, to be confirmed once those are added.
- R (for elevation data processing): `geodata`, `elevatr`

## Setup

1. Clone or download this repository.
2. Obtain the raw 2020 CPH microdata (PUF, organized by region/province) from
   PSA (see Data Access above) and place it under `PH_BEP/` following the
   folder structure expected by `do_files/data_construction/Process_2020popcenRaw.do`.
3. Run the data construction pipeline in order (see Structure below).
4. Open `do_files/master.do` and set the `parent` global to the path where
   you have placed this repository, e.g.:
   ```stata
   gl parent "/path/to/bep-philippines-replication"
   ```
5. Run `do_files/master.do` to reproduce the main analysis tables and figures.

## Structure

```
do_files/
    master.do                              Main analysis master do-file
    BLE_summary_stats_v2.do                Descriptive statistics
    BLE_summary_stats_children.do          Descriptive stats, intergenerational sample
    Popcen_estimate_mun.do                 Main DID estimates
    Santata_Chaismartin_dose_awareES_v6_1.do   Dose-aware event study (LDT)
    robustness_Q2_clean_vs_original.do     Robustness check (LDT quartile definition,
                                            "clean" vs. original Q2); called from
                                            master.do immediately after
                                            Santata_Chaismartin_dose_awareES_v6_1.do
    Intergenerational_effect_BEP_v2.do     Intergenerational transmission analysis
    data_construction/
        master_data_construction.do        Runs the full data construction pipeline in order
        Process_2020popcenRaw.do           Step 1: raw PSA CSV extracts -> merged .dta
        LDI_popcen2020_languagematch_muicipality.do
                                            Step 2a: municipality-level Linguistic
                                            Distance from Tagalog (LDT) index
        LDI_popcen2020_languagematch_province.do
                                            Step 2b: province-level Linguistic
                                            Distance from Tagalog (LDT) index
        Popcen2020_readyData_v3.do         Step 3: builds final analysis-ready dataset
                                            (merges LDT, elevation, and other
                                            municipality-level variables)
        Mapgraph_shapefiles.do             Step 4 (optional): produces map figures
                                            (Tagalog share, LDT quartile maps)
r_scripts/
    R_construct_elevation_geodata_PHIL.R   SRTM elevation control construction;
                                            run before Popcen2020_readyData_v3.do
output/
    figures/                               Figure output (generated by do-files)
    tables/                                Table output (generated by do-files)
```

### Data construction pipeline order

Run `do_files/data_construction/master_data_construction.do`, which executes
the following steps in order (see comments within for details):

1. `r_scripts/R_construct_elevation_geodata_PHIL.R` → `PH_BEP/LDI/terrain_ruggedness.csv`
   (run this first, separately, in R)
2. `Process_2020popcenRaw.do` → `POPCEN2020_f3_mmhh_merged.dta`
3. `LDI_popcen2020_languagematch_muicipality.do` → `PH_BEP/LDI/LDI_Tagalog_municipality.dta`
4. `LDI_popcen2020_languagematch_province.do` → `PH_BEP/LDI/LDI_Tagalog_birthprov.dta`
5. `Popcen2020_readyData_v3.do` → `PH_BEP/data/ReadyData/Popcen2020_readydata.dta` (final analysis dataset used by `../master.do`)
6. `Mapgraph_shapefiles.do` (optional; produces map figures only)

## Notes

- File paths in all do-files/scripts were converted from the authors' local
  environment (which used a mix of Dropbox and OneDrive as project roots
  across different files) to a unified relative structure for replication.
- Fixed a case-sensitivity filename mismatch (`popcen_estimate_mun.do` →
  `Popcen_estimate_mun.do`) that would have caused errors on case-sensitive
  filesystems (e.g., Linux, GitHub), though not on the authors' macOS
  environment.
- Fixed the same `TagSim=1 if lang=="Japanese"` bug in both
  `LDI_popcen2020_languagematch_muicipality.do` (municipality-level) and
  `LDI_popcen2020_languagematch_province.do` (province-level).
- Removed intentional "stop" markers (stray lines such as `s`/`ss`, used by
  the authors during development to halt execution partway through a
  do-file) and any incomplete/placeholder code that followed them.
